1 C 介绍

 

仙境传说(以下简称 RO) 游戏里面生命体的行动，是以RO Client 程序里设置的档案夹，以AI 档案夹里的AI.lua, Util.lua 为根据所操纵的。

RO玩家们可以制作出自己想要的人工智能，也可以使用别人做好的人工智能，也是主要目的。

 

剧本和RO Client是一种连动的程序，有文法上的错误的话是不会被执行的，有理论上的错误的话，也不能按照自己想要方式执行的。

 

RO开发部预计会提供，对剧本不需要做任何编辑，也可以玩游戏的好的剧本的。

 

预计要做到，对不太会使用程序的玩家们，也可以实现自己想要的人工智能做修正、补强的计划。

 

为了对程序上感兴趣的使用者着想，预计会陆陆续续的增加新的技能为计划的。

 

剧本所制造使用的语言是LUA，详情请看 (http://www.lua.org)

 

2- 剧本启动构造

 

RO Client是从Client程序，在生命体新形成的时机，以解读AI.lua, Util.lua档案后，才会启动人工智能的。

 

当生命体新形成的时机如下 : 

          1) 诞生生命体时，

       2) 把死亡的生命体复活时，

       3) 在角色选择窗口里，选择拥有生命体的角色后开始玩游戏时，

       4) 拥有生命体的角色，使用苍蝇翅膀、蝴蝶翅膀时，

       5) 拥有生命体的角色，使用传送之阵时，

       6) 拥有生命体的角色，使用卡普拉移动服务时，

       …

      

       上面事项的共同点是，当生命体在游戏空间里重新出现时，

 

解读完AI.lua档案后，RO Client程序从AI.lua 剧本里执行AI ( id)涵数， id是游戏上的生命体指定编号，id是RO Client传达给人工智能剧本的数字，可以把AI(id)的内容做编辑后，可以变化人工智能，基础的生命体行动，是指移动、攻击、喂食、使用技能等的函数，内建在RO Client 程序里，把自己所构成的适当的状况里，执行提供的函数即可。

 

Const.lua RO Client 程序内，里面有各种有关常数数值的情报，

Const.lua 请参考与 AI.lua Utiil.lua 即可。

Utiil.lua 请参考与 AI.lua 就可以了，目前在Util.lua 清单数据构造中，有几项单纯技能的计算函数在里面。

 

生命体人工智能必须的条件是，AI.lua 档案和AI.lua 档案里所定义的AI (id)函数，除了这两个条件之外都是选择事项，换句话说 Const.lua, Util.lua 也是选择的事项，不过在制作剧本时，还是需要各种情报的?

 

 

3- 仙境传说Client端 内建函数 说明

 

       # id : 游戏内的物体，所拥有的指定编号

 

1)  MoveToOwner (id)

id  : 生命体的 id

归还值: 没有

机能 : 让生命体移动到主人身边附近

 

2)  Move (id,x,y)

id  : 生命体的 id

                    x  : 目的地 横线 坐标

y  : 目的地 垂直 坐标

归还值 : 没有

机能 : 让生命体往目的地移动

 

3)  Attack (id1,id2)

id1 : 攻击者

id2 : 非攻击者

归还值 : 没有

机能 : 让生命体去攻击 id2者

 

4)  GetV (V_,id) end

V_... : 显示物体属性的常数数值

id  : 属性的对象

归还值 : V_... 依据V的数值做归换，举例来说 :在V_POSITION时，当x, y 坐标是 V_HP时，就是 HP 。

机能 : 获得 id 的属性 (V_...)，显示属性的常数数值在Util.lua里，已经定义了，

有关属性的详细内容 : 请参考‘4―仙境传说客户端 内藏常数值 说明’

 

5)  GetActors ()

归还值 : 是id们的集合，归换成lua的table型态

机能 : 在角色视线里的角色、NPC、魔物、物品、获得技能们的id

 

6)  GetTick ()

归还值 : 1/1000秒的，单位数字

机能 : 获得计算机的时间，这数值是指，计算机开始时从0开始，每1/1000秒 增加1，一直到目前的数值

 

7)  GetMsg (id)

 id  : 生命体的 id

归还值 : 从仙境传说Client传递的讯息，以lua的table型态归换

技能 : 把使用者得直接命令等，传给剧本

8)  GetResMsg (id)

id  : 生命体的 id

归还值 : 从仙境传说Client传递的预约讯息，以lua的table型态归换

机能 : 把使用者的直接预约命令等，传给剧本

      

10)  SkillObject (id,level,skill,target)

id : 生命体的 id

归还值 : 没有

机能 : id 所包含的东西是包含target的东西，有关level包含的数值是可使用的某种技能(skill)

 

11)  SkillGround (id,level,skill,x,y)

id 所包含的东西是包含x.y 的坐标，有关level所包含的数值是可使用的skill

 

13)  IsMonster (id)

id : 游戏内的物体

归还值 : id 所包含的东西是魔物的话可归换1，若不是的话可归换为0

机能 : 可判别魔物

 

       14) TraceAI (string)

string : 是纪录在TraceAI.txt 档案的内容，一定要文字文件

机能 : 纪录现在执行中的剧本，做为分析使用

举例来说，做完新的文字文件后，把文件名称改为 AI.lua 后，开启记事本，然后在空档案里输入

 

function        AI (myid)

 

end

 

写完之后，把之前有的AI 档案夹里的AI.lua 移动到别的地方或是更改名称后，把新制作的档案做复制，然后再执行RO Client就会启动，只是生命体什么都不做呆呆的站在那里。

 

 

5-2 有限状态 机械

 

要制造人工智能的方法有很多种，其中最单纯的使用最多的方法是，有限状态机械(FSM, Finite State Machine)，有关有限状态机械学术上的定义，书店里都有详细的介绍资料，这是大部份的人都知道的事实。

 

我们就拿我们的主题，生命体来举例好了，首先我们会期待我们的生命体会怎样的行动，

把这些东西先试着记下来好了，

 

寻找攻击主人的物体后，去攻击他，

周围有魔物的话先做攻击.. 或是闪躲 (会受伤地 ^^;;)

不要和主人距离太远，保持一定的距离

等等

 

按照状况会有很多的要求事项

 

假设我们已经把要求事项都整理好了的话，那我们就变成生命体想一下问题好了，

 

我们先想象一下周围没有魔物、主人也站着不动时，生命体也是休息的状态。

 

假设主人遭受到攻击或是我遭受到攻击的话，把攻击我的物体认定为敌人去追逐他，就会变成生命体会追逐某个物体的状态。

 

当敌人在攻击范围时，使用物理攻击或是使用技能时，就会变成和某个物体作战的状态。

 

当敌人已消灭了或是已逃到很远的地方时，就要回到主人的身边，这时候就会变成跟随主人的状态。

 

上面所讲的四种状态临时定义为 : 休息(IDLE)、追逐(CHASE)、攻击(ATTACK)、回归(FOLLOW) 的状态，

然后在个状态下发生某种变化时，会决定变成不是现在的状态而是别的状态，然后会指定这个状态时的动作，增加几种检查的条件后，把这个画成图画的话，如下所示 :

处理休息状况的函数定义为 OnIDLE_ST() ，并且让他在AI(myid) 函数里启动做为编辑。

 

function OnIDLE_ST ()

       local       object = GetOwnerEnemy (MyID)                            -- 寻找主人的敌人

       if (object ~= 0) then                                                 -- 有没有主人的敌人?

              MyState = CHASE_ST                                        -- 因为有主人的敌人转换成追逐状态             MyEnemy = object                                          -- 记忆我敌人的编号

              return                                                     -- 函数结束

       end

 

       object = GetMyEnemy (MyID)                                        -- 寻找我的敌人

       if (object ~= 0) then                                                 -- 有没有我的敌人?

              MyState = CHASE_ST                                        -- 因为有我的敌人转换成追逐状态

              MyEnemy = object                                      -- 记忆我敌人的编号

              return

       end

 

       local distance = GetDistanceFromOwner(MyID)                           -- 获得于主人的距离

       if ( distance > 3 or distance == -1) then                             -- 和主人的距离3空格以上，或是在主人的视线以外的话

              MyState = FOLLOW_ST                              -- 转换成回归状态

              return;

       end

end

 

 

function AI (myid)

MyID = myid

if (MyState == IDLE_ST) then

OnIDLE_ST ()

       end

end

 

当主人被攻击或是有我个敌人时，会转换成追逐的状态，若不附和这两个条件会计算出于主人的距离，若远离一定距离的话会转换成回归状态，

那么怎么会认知“主人被攻击了”，“有我的敌人”的条件呢?

 

所谓“主人被攻击了”的条件是，在周围里物体中的魔物，那对象对着主人时，若不是魔物而是一般玩家的角色来攻击主人时要怎么分辨呢? 只要去检查那角色的动作的话，就会知道的

 

function GetOwnerEnemy (myid)                                             -- 寻找主人的敌人

       local result = 0

       local owner  = GetV (V_OWNER,myid)                               -- 主人

       local actors = GetActors ()                                       -- 在主人视线里的物体们

       local enemys = {}                                              -- 负责主人敌人的table

       local index = 1                                                  

       local target

       for i,v in ipairs(actors) do                                               -- 把主人视线内的所有物体，全部视察

              if (v ~= owner and v ~= myid) then                            -- 某种物体的目标物

                     target = GetV (V_TARGET,v)

                     if (target == owner) then                                   -- 目标物是主人的话

                            if (IsMonster(v) == 1) then                  -- 某种物体是魔物的话

                                   enemys[index] = v                  -- 主人的敌人增加了

                                   index = index+1

                            else                                    -- 哪些物体不是魔物的话

                                   local motion = GetV(V_MOTION,i)    -- 哪些物体现在的动作

                                   if (motion == MOTION_ATTACK or motion == MOTION_ATTACK2) then C 若是攻击动作的话

                                          enemys[index] = v             -- 主人的敌人增加了

                                          index = index+1

                                   end

                            end

                     end

              end

       end

 

       local min_dis = 100                                           -- 最少的距离设定为100

       local dis

       for i,v in ipairs(enemys) do                                      -- 把主人的敌人们全部视察

              dis = GetDistance2 (myid,v)                              -- 主人和某个敌人的距离

              if (dis < min_dis) then                                       -- 距离少于最少距离时

                     result = v                                        -- 设定最终一个敌人

                     min_dis = dis                                        -- 设定最少距离

              end

       end

      

       return result                                                  -- 最终结果 归还 ( 0的话表示没有主人的敌人了)

end

 

结果的数值不是0时，表示主人正在被攻击中

 

 

所谓“有我的敌人” 的条件是怎么判断的呢?  假如想要让生命体先攻击魔物的话 ，假如周围有魔物的话先让判断成有敌人就可以啦，假如想要让生命体变成先被攻击后，再让生命体有反应的话，让生命体确认有没有攻击对象就可以啦，这两个选择决定，一个是选择先攻，和非先攻生命体的。

 

 

function GetMyEnemy (myid)                                           -- 寻找我的敌人

       local result = 0

 

       local type = GetV (V_HOMUNTYPE,myid)                          -- 我是哪一种类型生命体

5-5 文法错误检查和制作 TraceAI.txt 

 

完成的剧本档案，会拥有文法上的错误的，有文法错误的剧本，剧本在Client 程序解读时，会送出错误的讯息，

 

可以在还没有执行Client程序状态时，就可以先做第一次的文法检查，

在仙境传说里建置的档案夹里，点击两次 lua.exe档案的话，会出现命令proport

?入指令後 > dofile ‘./AI/AI.lua’

按Enter的话，假如有文法上的错误的话，会显示该错误的讯息，若没有错误的话不会显示任何讯息，

 

在这样子的检查里没有文法上的错误时，也会在生命体的状态下开始游戏时，会有出现错误讯息的情况，

这时候也是相同的移动到错误的行数上做修正后，重新再开始游戏，

 

虽然没有文法上的错误执行剧本后，即使生命体可以动作，有理论上的错误的话也会造成不能预期的行动，当碰到这样的时候，要把生命体的状态变化和各种数值的变化，做纪录后要做分析的必要性。

 

举例来说，生命体追逐到敌人身边后不做攻击的话，我们可以猜测也许在追逐的状态里， 有理论上的错误发生也不一定的。

 

function OnCHASE_ST ()

 

       TraceAI ("OnCHASE_ST")

 

       if (true == IsOutOfSight(MyID,MyEnemy)) then     -- ENEMY_OUTSIGHT_IN

              MyState = IDLE_ST

              MyEnemy = 0

              MyDestX, MyDestY = 0,0

              TraceAI ("CHASE_ST -> IDLE_ST : ENEMY_OUTSIGHT_IN")

              return

       end

       if (true == IsInAttackSight(MyID,MyEnemy)) then  -- ENEMY_INATTACKSIGHT_IN

              MyState = ATTACK_ST

              TraceAI ("CHASE_ST -> ATTACK_ST : ENEMY_INATTACKSIGHT_IN")

              return

       end

 

       local x, y = GetV (V_POSITION,MyEnemy)

       if (MyDestX ~= x or MyDestY ~= y) then                  -- DESTCHANGED_IN

              MyDestX, MyDestY = GetV (V_POSITION,MyEnemy);

              Move (MyID,MyDestX,MyDestY)

              TraceAI ("CHASE_ST -> CHASE_ST : DESTCHANGED_IN")

              return

       end

 

       TraceAI (string.format(“OnChase_ST end MyEnemy: %d, EnemyX : %d, EnemyY:%d, MyDestX:%d, MyDestY:%d\n”,MyEnemy,x,y,MyDestX,MyDestY))

 

end

 

如上面一样，想要知道特定状态处理函数的话，把想要知道的内容放在TraceAI 里面，

然后在游戏对话窗口里输入 /traceai的话，在仙境传说里设置的档案夹有TraceAI.txt 档案，会纪录那内容的，在重新输入/traceai 的话，这纪录就会停止， 

想要纪录某种变量数值时，使用string.format 可以修正文字行的
